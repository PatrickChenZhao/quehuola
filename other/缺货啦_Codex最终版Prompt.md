# “缺货啦”网站最终版 Codex Prompt

```text
你是一个资深前端工程师。请帮我生成一个完整可运行的网站项目。

项目名称：缺货啦

技术栈要求：
使用 Vite + React + TypeScript + Tailwind CSS。
这是一个纯前端单页应用，不需要后端、不需要登录、不需要数据库、不需要导出 Excel。
请生成结构清晰、组件化、可维护的代码。

网站定位：
“缺货啦”是一个服务于奶茶店员工的手机横屏缺货记录工具。用户通过图形化、写实插画风格的奶茶店场景，点击缺货物料，然后生成缺货清单。

核心要求：
1. 只适用于手机端横屏。
2. 不需要适配电脑端。
3. 用户竖屏打开网站时，不进入主界面，而是显示横屏提示页。
4. 横屏后进入主界面。
5. 主界面通过横向分页滑动切换三个区域。
6. 三个区域分别是：前台区、备料区、冰柜区。
7. 每个区域都是写实插画风格的场景原型。
8. 第一版不要依赖真实图片素材，用 CSS / SVG / div 图形搭建写实插画风格的场景结构，后续我可以替换成真实图片。
9. 物料图片和文字标签都可以点击。
10. 点击物料表示缺货，再次点击取消。
11. 点击“生成缺货表”后进入结果页。
12. 结果页显示表格和复制按钮。
13. 不保存记录，刷新或重新打开后重新开始。

整体视觉风格：
借鉴茶饮连锁门店的蓝白清爽、明亮、干净、现代奶茶店空间感。
只借鉴风格，不要出现任何茶百道 logo、茶百道文字、熊猫 IP 或任何真实品牌资产。
网站自己的品牌名称只显示“缺货啦”。

整体色彩：
- 主色调：蓝白清爽
- 辅助色：浅奶茶色、奶油色、浅棕色
- 强调色：红色，用于选中缺货状态
- 结果页使用浅奶茶色卡片和柔和圆角

字体和语言：
- 全部 UI 使用中文
- 字体使用系统默认中文字体即可
- 界面要清楚、适合门店员工快速操作

屏幕方向逻辑：
当用户竖屏打开网站时，显示一个居中的提示页面：
文案：“请将手机横屏使用缺货啦”
同时显示一个手机旋转小图标，可以用 CSS 或 SVG 绘制。
可以额外显示小字：“横屏后自动进入”
当屏幕切换为横屏后，自动进入主界面。

主界面顶部：
顶部采用极简样式，显示：
“缺货啦｜前台区”
“缺货啦｜备料区”
“缺货啦｜冰柜区”
根据当前区域动态变化。
不要显示已选数量。

主界面分页和滑动逻辑：
主界面是横屏手机端的循环分页场景。
默认进入后显示“前台区”。

区域顺序：
前台区 → 备料区 → 冰柜区 → 前台区，循环切换。

交互方向：
用户手指向右滑动时，视为镜头向左转，依次从：
前台区 → 备料区 → 冰柜区 → 前台区。
也支持反向滑动：
用户手指向左滑动时，反向切换：
前台区 → 冰柜区 → 备料区 → 前台区。

转场效果：
使用横向分页滑动 + 吸附效果 + 流畅过渡。
转场采用轻微 3D 空间感：
- 横向滑动
- 轻微旋转
- 轻微缩放
- 轻微淡入淡出
视觉上像用户站在奶茶店中间，镜头转向不同区域，而不是普通列表页面。
实现可以使用 carousel / CSS transform / snap slider，但需要循环分页。

主界面分页提示：
底部或顶部附近显示 3 个小圆点页码提示：
前台区：● ○ ○
备料区：○ ● ○
冰柜区：○ ○ ●

主按钮：
右下角固定一个悬浮按钮：
“生成缺货表”
按钮始终可见。
按钮风格要醒目，但与整体奶茶店风格协调。

物料标签样式：
每个物料旁边有奶茶色圆角标签。
标签背景：浅奶茶色 / 奶油色。
文字颜色：深棕色。
标签和物料图形都可以点击。
选中后不要改变成很夸张的红底，可以让标签文字略微变深或变红，但重点效果在物料图形上。

物料选中效果：
点击物料后表示缺货。
选中状态效果：
- 物料外面出现红色描边
- 物料轻微放大
- 物料轻微提亮
- 效果要有 transition 动画
再次点击取消选中。

数据结构要求：
请在代码里用 TypeScript 定义清晰的数据结构，例如：
Area
MaterialItem
selectedItems
每个物料至少包含：
id
name
area
displayOrder
position 或 layout 信息
category 可选

结果页排序：
结果页表格顺序固定为：
1. 前台区
2. 备料区
3. 冰柜区
每个区域内部按照场景摆放顺序显示，而不是按照点击顺序显示。

复制文字版本顺序：
复制文字也按固定顺序：
前台区
备料区
冰柜区

如果某个区域没有缺货物料，则隐藏该区域。
如果没有任何缺货物料，则复制内容为：
“啥也不缺”

结果页：
点击“生成缺货表”后，进入整页结果页，不要弹窗。
结果页风格延续茶饮店气质：
- 浅奶茶色卡片
- 柔和圆角
- 干净明亮
- 表格清晰

结果页内容：
如果有缺货物料，显示表格。
表格只显示两列：
区域
物料名称

每一行右侧有一个 “×” 删除按钮。
点击 “×” 后，直接删除该行，不需要二次确认。
删除后同步取消该物料的选中状态。
用户点击“返回继续修改”后，主界面中该物料也应该是未选中状态。

如果没有选择任何物料就点击“生成缺货表”，仍然进入结果页，显示：
“啥也不缺”

结果页按钮：
1. 复制文字版本
2. 返回继续修改

点击“复制文字版本”后：
复制按区域分组的缺货清单。
复制成功后弹出提示：
“已复制缺货清单”

复制格式示例：
缺货清单：
前台区：中pp杯、粗吸管、杯盖
备料区：小西米、黑糖粉、椰果
冰柜区：牛奶、淡奶油、青提汁

如果没有任何缺货项，复制内容为：
啥也不缺

点击“返回继续修改”：
返回主界面。
保留之前已选内容。
如果结果页删除过某些物料，返回后这些物料应该保持未选中。

不要添加：
- 不要登录
- 不要数据库
- 不要导出 Excel
- 不要主界面的清空已选按钮
- 不要显示已选数量
- 不要长期保存记录
- 不要使用 localStorage 保存选择状态
- 不要出现真实茶百道品牌 logo 或文字

三个区域的详细场景与物料如下：

- 横屏场景
- 白色台面、白色柜身的长柜台
- 背景是店内朝向外面大门的真实空间感
- 门外部分景深虚化
- 整体像真实奶茶店前台
- 风格是 CSS / SVG / div 组成的写实插画场景原型

前台区物料：
1. 中pp杯
2. 大pp杯
3. 中纸杯
4. 大纸杯
5. 粗吸管
6. 细吸管
7. 杯套
8. 杯盖
9. 纸巾
10. 单杯袋子
11. 双杯袋子
12. 四杯袋子
13. 保温单杯
14. 保温双杯
15. 保温四杯
16. 双杯托
17. 四杯托

前台区布局：
1. 红框区域是吸管 / 杯盖架：
   - 上层：粗吸管、细吸管
   - 下层：杯盖

2. 绿色区域是包装袋架：
   - 单杯袋子
   - 双杯袋子
   - 四杯袋子

3. 黄色区域是杯子架：
   - 4 个竖向分区
   - 中pp杯
   - 大pp杯
   - 中纸杯
   - 大纸杯

4. 纸巾：
   - 放在台面左侧小盒里

5. 杯套：
   - 放在侧立架

6. 保温单杯 / 保温双杯 / 保温四杯：
   - 放在柜台下层

7. 双杯托 / 四杯托：
   - 放在保温杯下一层

前台区的物料都要有可点击区域和奶茶色文字标签。

  1. 最左侧绿色区域：抽拉盒架子
  2. 中间黑色区域：主货架
  3. 右侧黄色区域：窄层架

备料区物料：

8. 冻冻粉
9. 糖

20. 手套

   - 是一个高柜 / 立架

   - 抽屉是白色透明、略带磨砂质地
   - 柜子的外框是黑色塑料
   - 存放：小西米

2. 中间黑色主货架：
   - 是黑色货架
   - 一共 4 层
   - 当前画面主要聚焦下面 3 层
   - 第 4 层只露出一点画面，放满杯子作为背景

3. 中间黑色货架的茶叶层：
   - 放五种茶叶
   - 茶叶是银色反光材质的袋装茶
   - 包括：
     - 焙香乌龙茶

========================
组件建议
========================

请至少拆分以下组件：
- App
- OrientationGate 或 RotateHint
- MainScene
- AreaCarousel
- AreaHeader
- PaginationDots
- FloatingGenerateButton
- MaterialHotspot
- FrontCounterScene
- PrepAreaScene
- FreezerAreaScene
- ResultPage
- ResultTable
- Toast

状态管理：
可以使用 React useState。
不需要 Redux。
不需要 localStorage。
selectedItems 可以用 Set<string> 或 string[] 管理。

交互细节：
1. MaterialHotspot 负责物料点击。
2. 每个物料有自己的 id。
3. 选中状态由 selectedItems 判断。
4. 选中后增加 selected class。
5. 删除表格行时，从 selectedItems 移除对应 id。
6. 返回继续修改时不要重置状态。
7. 刷新页面后状态自然重置。

滑动实现：
请实现触摸滑动：
- touchstart 记录起点
- touchend 判断滑动方向
- 支持循环切换
- 右滑进入下一个区域：前台区 → 备料区 → 冰柜区 → 前台区
- 左滑进入上一个区域：前台区 → 冰柜区 → 备料区 → 前台区
也可以同时支持点击小圆点切换区域，但不是必须。
转场使用 CSS transform 和 transition，加入轻微 3D 旋转、缩放、透明度变化。
切换后自动吸附到当前区域。

样式要求：
- 使用 Tailwind CSS
- 可以配合少量自定义 CSS
- 所有主要元素都要有圆角、阴影、柔和过渡
- 不要让标签过度遮挡物料
- 物料标签要清楚可读
- 横屏手机下优先保证点击区域足够大
- 场景原型要有层次感，不要做成普通列表

空状态：
结果页没有选择任何物料时：
显示大字：
“啥也不缺”
可以附加一行小字：
“今天物料很充足”
但主要文案必须是“啥也不缺”。

复制功能：
使用 navigator.clipboard.writeText。
复制成功显示 Toast：
“已复制缺货清单”
如果浏览器不支持 clipboard，可以降级使用临时 textarea 复制。

请生成完整项目代码：
- package.json
- vite 配置
- tsconfig
- Tailwind 配置
- src 目录
- React 组件
- 样式
- 示例数据
- README 简要说明如何运行

运行方式：
npm install
npm run dev

最终目标：
生成一个可以直接运行的手机横屏网页原型。它应该能完成：
1. 竖屏提示横屏使用
2. 横屏进入主界面
3. 循环滑动切换前台区、备料区、冰柜区
4. 点击物料选择缺货
5. 选中物料有红色描边、轻微放大、轻微提亮
6. 点击生成缺货表进入结果页
7. 表格显示区域和物料名称
8. 每行可以用 × 删除
9. 可以复制文字版本
10. 可以返回继续修改
11. 不保存历史记录
```
